# AGENTS.md — Контекст для AI-агентов

## Обзор проекта

Это коллекция скриптов на **AutoHotkey v2** для Windows, превращающая стандартную периферию в профессиональный инструмент управления. Проект оптимизирован для фронтенд-разработки, навигации в стиле macOS и кастомизации вертикальных мышей (Ugreen M571P и аналоги).

Проект не использует пакетные менеджеры, системы сборки или фреймворки. Это чистые `.ahk`-скрипты, интерпретируемые средой выполнения AutoHotkey v2. Никаких файлов конфигурации уровня проекта (вроде `package.json`, `pyproject.toml`, `Cargo.toml` и т.д.) не существует.

## Структура проекта

```
.
├── index.ahk           ; Точка входа. Подключает все модули через #Include
├── macos_logic.ahk     ; Слой навигации через Caps Lock (Tap vs Hold)
├── language_switch.ahk ; Умное переключение раскладки клавиатуры через Shift
├── vertical_mouse.ahk  ; Логика боковых кнопок мыши и контекстных действий
├── unplug_insert.ahk   ; Полное подавление клавиши Insert
├── README.md           ; Человеко-читаемая документация
└── AGENTS.md           ; Этот файл
```

### Назначение файлов

- **`index.ahk`** — единственный файл, который нужно запускать. Содержит директивы `#Requires AutoHotkey v2.0`, `#SingleInstance Force`, `SetWorkingDir A_ScriptDir` и последовательно подключает все остальные модули через `#Include`.
- **`macos_logic.ahk`** — переопределяет Caps Lock в модификатор слоя. Содержит `#Requires AutoHotkey v2.0`, `#SingleInstance Force`, `InstallKeybdHook` и `A_MenuMaskKey := "vkE8"`. Короткое нажатие (< 200 мс) переключает Caps Lock; удержание активирует навигацию:
  - `CapsLock + Left/Right` → `Home` / `End`
  - `CapsLock + Up/Down` → `Ctrl+Home` / `Ctrl+End`
  - `CapsLock + Shift + стрелки` → выделение (`+{Home}`, `+{End}`, `^+{Home}`, `^+{End}`)
  - `CapsLock + Backspace` → удаление до начала строки (`+{Home}{Backspace}`)
- **`language_switch.ahk`** — переключает раскладку клавиатуры одиночным нажатием Shift. Содержит `#Requires AutoHotkey v2.0` (после закомментированного устаревшего блока с LCtrl/RCtrl). Использует `A_PriorKey` для проверки, что Shift был нажат одиночно, а не в комбинации:
  - `~LShift Up` (одиночное нажатие) → Русский (`RU := 0x4190419`)
  - `~RShift Up` (одиночное нажатие) → Английский (`EN := 0x4090409`)
  - Переключение выполняется через `PostMessage(0x50, 0, LayoutID, ...)` (`WM_INPUTLANGCHANGEREQUEST`) в активное окно или элемент с фокусом ввода.
- **`vertical_mouse.ahk`** — логика боковых кнопок мыши (XButton1, XButton2). **Не содержит** `#Requires AutoHotkey v2.0` (полагается на точку входа). Использует `GroupAdd` для группировки приложений:
  - Группа `Editors`: `Code.exe`, `zed.exe`
  - Группа `Browsers`: `chrome.exe`, `vivaldi.exe`, `msedge.exe`, `firefox.exe`
  - Глобальные хоткеи:
    - `*XButton1` → предыдущий виртуальный рабочий стол (`^#{Left}`)
    - `*XButton2` → следующий виртуальный рабочий стол (`^#{Right}`)
    - `#!XButton1` / `#!XButton2` → блокировка Windows (`DllCall("LockWorkStation")`)
  - Контекстные хоткеи (`Ctrl + боковая кнопка`):
    - В редакторах: `^XButton1` → AI Chat (`^!b`), `^XButton2` → Терминал (`^j`)
    - В браузерах: `^XButton1` → Mobile View (`^+m`), `^XButton2` → DevTools (`{F12}`)
- **`unplug_insert.ahk`** — однострочный скрипт (`Insert::return`), полностью подавляющий клавишу Insert. **Не содержит** `#Requires AutoHotkey v2.0`.

## Технологический стек

- **Язык**: AutoHotkey v2 (синтаксис v2-native, без устаревших конструкций v1)
- **Среда выполнения**: AutoHotkey v2 для Windows
- **API**: WinAPI вызовы через `DllCall`, сообщения Windows через `PostMessage`, хуки клавиатуры (`InstallKeybdHook`)
- **Группировка окон**: `GroupAdd` + `ahk_group` для применения хоткеев к категориям приложений
- **Маскировка меню**: `A_MenuMaskKey := "vkE8"` предотвращает появление меню Пуск при использовании модификаторов в хоткеях

## Сборка и запуск

### Требования
- Установленная среда [AutoHotkey v2](https://www.autohotkey.com/)

### Запуск
```powershell
# Прямой запуск
.\index.ahk
```
Или двойной клик по `index.ahk` в проводнике.

### Автозагрузка
Для автоматического запуска при входе в Windows создайте ярлык файла `index.ahk` в папке `shell:startup` (можно открыть через `Win + R` → `shell:startup`).

### Нет процесса сборки
Проект не требует компиляции, транспиляции или сборки. Все файлы — исходный код, который интерпретируется напрямую. Нет скриптов CI/CD, Dockerfile, Makefile или аналогичных инструментов.

## Стиль кода и соглашения

### Общие правила
- Основные модули (`index.ahk`, `macos_logic.ahk`, `language_switch.ahk`) начинаются с `#Requires AutoHotkey v2.0`
- Используется современный синтаксис v2: фигурные скобки `{}` для блоков, отсутствие устаревших команд в одну строку
- Комментарии и документация ведутся на **русском языке**
- Горячие клавиши с префиксом `*` (например, `*CapsLock::`, `*XButton1::`) игнорируют зажатые модификаторы
- `~` перед хоткеем (например, `~LShift Up::`) позволяет пропустить оригинальное действие клавиши, чтобы не нарушить стандартное поведение Shift при зажатии
- Условные хоткеи оборачиваются в `#HotIf` / `#HotIf` (современный аналог `#IfWinActive` в v2)

### Группировка процессов
При добавлении нового приложения в существующую группу используйте `GroupAdd`:
```autohotkey
GroupAdd("GroupName", "ahk_exe process.exe")
```
Для проверки активного окна из группы:
```autohotkey
#HotIf WinActive("ahk_group GroupName")
    ; хоткеи для группы
#HotIf
```

### Переключение раскладки
Используется HKL (Handle to Keyboard Layout) в формате `0xLLLLRRRR`, где младшее слово — язык, старшее — раскладка:
- `EN := 0x4090409` — Английский (US)
- `RU := 0x4190419` — Русский

Переключение выполняется через `PostMessage(0x50, 0, LayoutID, ...)` (сообщение `WM_INPUTLANGCHANGEREQUEST`). Функция `SetLayout(LayoutID)` определяет наличие фокуса ввода через `ControlGetFocus` и направляет сообщение либо конкретному элементу управления, либо самому окну.

## Тестирование

Проект **не содержит автоматизированных тестов**. Тестирование выполняется вручную:

1. Запустить `index.ahk`
2. Проверить каждый слой:
   - **Caps Lock**: короткое нажатие (< 200 мс) переключает индикатор; удержание + стрелки перемещает курсор (Home/End/Ctrl+Home/Ctrl+End); удержание + Shift + стрелки выделяет текст; удержание + Backspace удаляет до начала строки
   - **Shift**: одиночное нажатие Left Shift переключает на русский, Right Shift — на английский; комбинации типа `Shift+A` или `Shift+стрелка` не должны вызывать смену языка
   - **Мышь**: боковые кнопки переключают виртуальные рабочие столы; `Ctrl + боковая кнопка` в VS Code/Zed и браузерах вызывает контекстные действия; `Win + Alt + боковая кнопка` блокирует рабочую станцию
   - **Insert**: клавиша не должна ничего делать

## Безопасность

- Скрипт использует `DllCall("LockWorkStation")` для блокировки Windows. Это стандартный системный вызов WinAPI.
- `PostMessage` с `WM_INPUTLANGCHANGEREQUEST` (0x50) отправляется только в активное окно (`WinActive("A")`) или в элемент с фокусом ввода (`ControlGetFocus`).
- `A_MenuMaskKey := "vkE8"` подавляет появление меню Пуск при использовании модификаторов в хоткеях (vkE8 — неиспользуемый виртуальный код клавиши).
- `#SingleInstance Force` гарантирует, что при повторном запуске `index.ahk` предыдущий экземпляр скрипта будет перезапущен, а не запущен параллельно.
